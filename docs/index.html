"FastAPI on EC2, PostgreSQL on RDS, Redis on ElastiCache for conversation state and idempotency, S3 for media. Each clinic gets an isolated webhook route with an AES-256-GCM encrypted bot token — multi-tenancy enforced at the database layer from day one."

Multi-tenancy

Every clinic has its own webhook endpoint, encrypted token, and data isolation. One deployment, unlimited clinics.

Idempotency

Redis-keyed deduplication prevents duplicate bookings under Telegram's retry behavior. Conversation state survives process restarts.

Security

Per-clinic webhook secrets, CSRF-protected dashboard routes, HttpOnly session cookies, and no bot tokens stored in plaintext.
